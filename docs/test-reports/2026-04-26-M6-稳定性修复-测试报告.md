## 3. 问题发现

### Issue #1 — Info: fetchPlaylistDetailV2 模块名未实测验证

- **严重程度**：Info
- **位置**：`qq/qq.go:491` — `"music.srfDissInfo.DissInfo"`
- **描述**：V2 接口的 musicu.fcg 模块名来自社区逆向，无官方文档。当前测试通过说明旧接口对测试用例中的 content_id 仍然有效（未触发 fallback）。V2 路径尚未在真实场景中被触发验证。
- **建议**：在 NAS 部署后，手动测试一个旧接口确实返回空 cdlist 的推荐歌单 ID，确认 V2 fallback 正常工作。

### Issue #2 — Minor: fetchPlaylistDetailV2 歌曲上限 100 首

- **严重程度**：Minor
- **位置**：`qq/qq.go:497` — `"song_num": 100`
- **描述**：V2 接口请求最多 100 首歌曲。如果推荐歌单超过 100 首，只返回前 100 首。旧接口无此限制。
- **影响**：部分大型歌单可能不完整。但推荐歌单通常 20-50 首，实际影响低。
- **建议**：当前可接受，如后续反馈有截断再增加分页。

### Issue #3 — Info: canRecommend 在 capabilities 未加载前为 false

- **严重程度**：Info
- **位置**：`frontend/src/views/SearchView.vue:22-26`
- **描述**：`capabilities` 由 `App.vue` 的 `initApp()` 异步加载。在加载完成前，`canRecommend` 为 false，推荐按钮不显示。加载完成后 Vue 响应式更新会自动显示。
- **影响**：极快网络下用户可能不会注意。慢网络下推荐按钮有短暂延迟出现。
- **建议**：可接受行为，不需要修复。
