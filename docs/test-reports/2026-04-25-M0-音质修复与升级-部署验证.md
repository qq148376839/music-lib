# 部署验证报告: M0 音质修复与升级

> 日期: 2026-04-25
> Dockerfile: 3 阶段构建（Node → Go → Alpine）
> docker-compose: 单服务 + health check
> 验证方式: 静态分析（本机无 Docker daemon）

---

## 总体结论: 不通过

发现 1 个 Critical 问题阻塞部署。M0 前端改动在 vanilla JS 文件中，但 Docker 容器只嵌入并服务 Vue 前端（web/dist/）。NAS 上看不到任何 M0 前端功能。

---

## Critical-1: M0 前端改动未进入 Docker 镜像

**问题**:

项目存在两套前端：

| 前端 | 文件位置 | Docker 中的状态 |
|------|---------|----------------|
| Vue 3 SPA | `frontend/` → 编译到 `web/dist/` | 被嵌入到 Go 二进制，**是 NAS 实际服务的 UI** |
| Vanilla JS | `web/index.html`, `web/js/app.js`, `web/css/style.css` | **不在 `web/dist/` 中，未嵌入，NAS 不可见** |

M0 的前端改动全部在 vanilla JS 中：
- `web/js/app.js` — 音质 tag、降级提示、升级按钮、全部升级
- `web/css/style.css` — quality-tag、upgrade-bar 样式
- `web/index.html` — favicon 引用、upgrade-bar 容器
- `web/favicon.svg` — favicon 图标

Docker 构建链路：
```
Dockerfile Stage 1: frontend/ → npm run build → web/dist/ (Vue)
Dockerfile Stage 2: go build → 嵌入 web/dist/ (via go:embed)
```

`web/js/app.js` 等 vanilla 文件从未进入 `web/dist/`，也未被 `go:embed` 包含。

**影响**: NAS 部署后，用户看不到：
- 实际音质显示
- 降级提示
- 升级按钮 / 全部升级
- favicon

后端 API（`POST /api/nas/download/upgrade`、Task 新字段）正常工作，但无前端调用。

**Vue 前端（DownloadView.vue）现状**:
- 有任务列表 ✓
- 有批次汇总 ✓
- **无** actual_quality 显示
- **无** 降级/升级标识
- **无** 升级按钮
- **无** favicon
- **无** upgrade API 调用

**修复建议**: 将 M0 前端功能移植到 Vue 前端 `frontend/src/views/DownloadView.vue`。需要：
1. DownloadView.vue 增加音质 tag、降级/升级标识
2. 增加单首升级按钮和全部升级按钮
3. `frontend/src/api.js` 增加 `upgradeDownloads` 方法
4. `frontend/index.html` 或 Vue App.vue 引用 favicon
5. 重新 `npm run build` 生成 `web/dist/`

### Minor-1: Dockerfile 删除 package-lock.json 后重新安装

**位置**: Dockerfile 第 4-5 行

```dockerfile
COPY frontend/package.json frontend/package-lock.json* ./
RUN rm -f package-lock.json && npm config set registry https://registry.npmmirror.com && npm install
```

先 COPY package-lock.json，然后 rm -f 删掉，再 `npm install`（不是 `npm ci`）。这意味着每次构建可能安装不同版本的依赖。

**影响**: 构建不可复现。

**建议**: 保留 lock 文件，使用 `npm ci`。

### Minor-2: favicon.svg 未进入 web/dist/

**位置**: `web/favicon.svg`

favicon 文件在 `web/` 根目录，不在 `web/dist/`。Go embed 只嵌入 `dist/` 子目录，favicon 不会被包含在 Docker 镜像中。

**修复**: 要么在 Vue 的 `public/` 目录放一份 favicon，让 Vite build 自动复制到 dist/；要么在 Dockerfile 中手动复制。

### Info-1: 缺少内存限制

NAS（如群晖 DS220+）通常只有 2GB RAM。建议在 docker-compose 中加 `mem_limit: 512m` 防止 OOM。

## 结论

**后端可部署，前端阻塞。**

M0 后端改动（Provider 回写、Writer 重构、升级 API）已通过编译和测试，Docker 构建不受影响。

但 M0 前端改动全部在 vanilla JS 文件中，Docker 镜像只嵌入 Vue 前端（`web/dist/`），NAS 上无法看到任何 M0 前端功能。

**下一步**: 将 M0 前端功能移植到 Vue 前端（`frontend/src/views/DownloadView.vue`），重新 build dist/，然后部署。
