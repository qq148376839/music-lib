# Milestone 4 — Web UI 升级

> 状态：待开始
> 前置条件：Milestone 1-3 全部完成（API 稳定后才做 UI）
> 预计工作量：中（API 已有，前端纯展示逻辑）

---

## 这个 Milestone 解决什么问题

M1-M3 完成后，所有功能必须通过 API 调用才能使用，监控功能完全没有 UI 入口。现有 vanilla HTML/JS 界面能搜索和下载，但：
- 没有下载历史页面
- 没有监控管理页面
- 质量和登录状态 UX 较粗糙

这个 Milestone 完成后：NAS 用户通过浏览器完成所有操作，不需要懂 API。

---

## 交付范围

### 技术栈

- Vue 3 + Vite（替换现有 vanilla JS）
- Naive UI 组件库
- `go:embed` 将 dist/ 嵌入二进制

### 页面结构（4 个主页面）

**1. 搜索页（现有功能优化）**
- 平台选择（下拉或 Tab）
- 搜索结果列表，含封面缩略图
- 单曲下载（浏览器 + NAS 两个按钮）
- 歌单链接解析入口

**2. 下载队列页**
- 活跃任务（进行中、等待中）
- 历史记录（完成/失败，分页）
- 失败任务重试
- 批量任务折叠展示

**3. 监控管理页**（M3 的 UI 入口）
- 监控列表（启用/禁用切换）
- 新建监控（平台选榜单、TopN 滑块、间隔选择）
- 执行历史（最近 10 次，含统计）
- 手动立即触发按钮

**4. 设置页**
- 登录状态（网易云 + QQ 音乐，QR 码扫码）
- 全局音质选择
- NAS 目录路径显示（只读，来自配置）

### 构建集成

```
Dockerfile 更新为三阶段：
  Stage 1: node:20-alpine → npm run build → dist/
  Stage 2: golang:1.22-alpine → go build（含 embed dist/）
  Stage 3: alpine:3.20 → 最终镜像
```

## 验收条件

以下全部满足才算完成：

1. 访问 `localhost:35280` 看到 Vue 3 UI，搜索一首歌并下载成功
2. 下载队列页能看到历史记录（M1 SQLite 持久化的数据）
3. 监控管理页能完成：新建监控 → 手动触发 → 看到执行历史
4. QR 码登录流程在 UI 内完整可用
5. `docker-compose up --build` 单命令完成，镜像 < 60MB
